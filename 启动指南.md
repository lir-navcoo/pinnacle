## 🚀 快速启动

### 方式一：开发模式（推荐）

#### 1. 启动后端服务

```bash
cd /Users/lirui/Documents/pinnacle/backend

# 确保依赖已安装
uv sync

# 启动后端（端口 8000）
uv run uvicorn app.main:app --reload --port 8000
```

**访问 API 文档：** http://localhost:8000/docs

#### 2. 启动前端服务

在新终端窗口：

```bash
cd /Users/lirui/Documents/pinnacle/frontend

# 启动开发服务器（端口 3000）
npm run dev
```

**访问前端界面：** http://localhost:3000

## ⚙️ 环境配置

### 后端环境变量

创建 `backend/.env` 文件（基于 `.env.example`）：

```bash
# 数据库配置
DATABASE_URL=postgresql+asyncpg://postgres:postgres@localhost:5432/pinnacle

# Redis 配置
REDIS_URL=redis://localhost:6379/0

# JWT 密钥（生产环境请修改）
SECRET_KEY=your-secret-key-change-in-production

# 阿里云 OSS（可选）
OSS_ENDPOINT=https://oss-cn-hangzhou.aliyuncs.com
OSS_ACCESS_KEY_ID=your-access-key
OSS_ACCESS_KEY_SECRET=your-access-secret
OSS_BUCKET_NAME=pinnacle
OSS_REGION=cn-hangzhou

# 钉钉机器人（可选）
DINGTALK_WEBHOOK_URL=
DINGTALK_SECRET=
```

### 数据库初始化

如果首次运行，需要创建数据库：

```bash
# 使用 psql 创建数据库
psql -U postgres
CREATE DATABASE pinnacle;
\q
```

## 🔧 常见问题

### 1. 后端启动失败：Address already in use

```bash
# 关闭占用 8000 端口的进程
lsof -ti:8000 | xargs kill -9
```

### 2. 后端启动失败：No module named 'xxx'

```bash
cd backend
uv sync
```

### 3. 前端启动失败：Module not found

```bash
cd frontend
npm install
```

### 4. 数据库连接失败

确保 PostgreSQL 正在运行：

```bash
# macOS 使用 brew 安装 PostgreSQL
brew services start postgresql

# 或者手动启动
pg_ctl -D /usr/local/var/postgres start
```
