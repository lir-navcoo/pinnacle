# Pinnacle 项目配置说明

## 📋 配置更新概览

本次更新将项目从 PostgreSQL + 阿里云 OSS 改为：
- ✅ **MySQL** 数据库（本地）
- ✅ **Redis** 缓存（本地）
- ✅ **本地文件存储** 替代阿里云 OSS

---

## 🗄️ 数据库配置

### 1. 创建 MySQL 数据库

```bash
# 登录 MySQL
mysql -u root -p

# 执行初始化脚本
source /Users/lirui/Documents/pinnacle/backend/init_db.sql
```

或者手动创建：

```sql
CREATE DATABASE IF NOT EXISTS pinnacle 
DEFAULT CHARACTER SET utf8mb4 
DEFAULT COLLATE utf8mb4_unicode_ci;
```

### 2. 配置数据库连接

编辑 `backend/.env` 文件（或复制 `env.example`）：

```bash
DATABASE_URL=mysql+aiomysql://root:root@localhost:3306/pinnacle
```

**参数说明：**
- `root:root` - 用户名和密码，请根据实际情况修改
- `localhost:3306` - MySQL 主机和端口
- `pinnacle` - 数据库名

---

## 🔧 Redis 配置

Redis 默认配置即可，无需额外设置：

```bash
REDIS_URL=redis://localhost:6379/0
```

确保本地 Redis 服务正在运行：

```bash
# macOS 使用 brew
brew services start redis

# 或直接启动
redis-server
```

## 📁 本地文件存储配置

### 1. 配置存储类型

在 `backend/.env` 中设置：

```bash
STORAGE_TYPE=local
LOCAL_STORAGE_PATH=/Users/lirui/Documents/pinnacle/backend/uploads
LOCAL_STORAGE_URL_PREFIX=/uploads
```

### 2. 目录结构

上传的文件会按日期分层存储：

```
backend/uploads/
└── pinnacle/
    └── 2024/
        └── 03/
            └── 26/
                └── 143052_abc123.png
```

### 3. 访问 URL

生成的图片访问路径：

```
http://localhost:8000/uploads/pinnacle/2024/03/26/143052_abc123.png
```

## 🚀 快速启动

### 步骤 1：安装依赖

```bash
cd /Users/lirui/Documents/pinnacle/backend
uv sync
```

### 步骤 2：配置环境变量

```bash
cd /Users/lirui/Documents/pinnacle/backend
cp env.example .env
# 编辑 .env 文件，配置数据库连接
```

### 步骤 3：初始化数据库

```bash
# 方式一：使用 SQL 脚本
mysql -u root -p < init_db.sql

# 方式二：手动创建数据库
mysql -u root -p
> CREATE DATABASE pinnacle;
```

### 步骤 4：启动后端服务

```bash
cd /Users/lirui/Documents/pinnacle/backend
uv run uvicorn app.main:app --reload --port 8000
```

访问 http://localhost:8000/docs 查看 API 文档

### 步骤 5：启动前端服务（新终端窗口）

```bash
cd /Users/lirui/Documents/pinnacle/frontend
npm run dev
```

访问 http://localhost:3000

## ⚙️ 高级配置

### 切换到阿里云 OSS（可选）

如果后续想使用阿里云 OSS，只需修改配置：

```bash
STORAGE_TYPE=oss
OSS_ENDPOINT=https://oss-cn-hangzhou.aliyuncs.com
OSS_ACCESS_KEY_ID=your_access_key
OSS_ACCESS_KEY_SECRET=your_access_secret
OSS_BUCKET_NAME=pinnacle
OSS_REGION=cn-hangzhou
```

### MySQL 性能优化

对于生产环境，建议优化 MySQL 配置：

```ini
[mysqld]
max_connections=200
innodb_buffer_pool_size=1G
innodb_log_file_size=256M
query_cache_size=64M
```

## 🔍 常见问题

### 1. 数据库连接失败

```bash
# 检查 MySQL 是否运行
mysql.server status

# 检查数据库是否存在
mysql -u root -p -e "SHOW DATABASES LIKE 'pinnacle';"

# 检查用户权限
mysql -u root -p -e "SHOW GRANTS FOR 'root'@'localhost';"
```

### 2. 文件上传目录无权限

```bash
# 确保 uploads 目录存在且有写权限
mkdir -p /Users/lirui/Documents/pinnacle/backend/uploads
chmod 755 /Users/lirui/Documents/pinnacle/backend/uploads
```

### 3. Redis 连接失败

```bash
# 检查 Redis 是否运行
redis-cli ping
# 应该返回 PONG

# 重启 Redis
brew services restart redis
```
